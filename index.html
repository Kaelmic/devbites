Why “10” is not equal to 10

user_input = input("Enter a number: ")

if user_input == 10:
    print("Correct")
else:
    print("Wrong")

What happened? Input returns a string, not an integer. That means "10" and 10 are different types.

1. Why input() breaks your comparison

Click other cards to switch

The input() function always returns text. So if you compare it directly to a number, the condition can fail or even throw an error depending on the operator you use. 

age = input("Enter age: ")

if age >= 18:
    print("Adult")
Common mistake: Forgetting to convert the input first with int().

Better version

age = int(input("Enter age: "))

if age >= 18:
    print("Adult")
  • input() always returns a string.
  • Comparing string to int causes problems.
  • Converting early keeps the rest of your logic clean.
