Why “10” is not equal to 10

user_input = input("Enter a number: ")

if user_input == 10:
    print("Correct")
else:
    print("Wrong")

What happened? Input returns a string, not an integer. That means "10" and 10 are different types.

2. Why your list keeps remembering values

Python Real-world confusion

Mutable default arguments can reuse the same list between function calls. It feels like Python is “remembering” old data because it kind of is. 

def add_item(item, my_list=[]):
    my_list.append(item)
    return my_list
Common mistake: Using a list as a default parameter instead of starting with None.

3. Why “10” is not equal to 10

Python Type basics

Values can look the same but still be different because one is text and the other is a number. This is one of the most common beginner traps. 

print("10" == 10)   # False
print(int("10") == 10)   # True
Common mistake: Assuming what the user typed is automatically numeric.

4. The SQL mistake that can destroy data

SQL Workplace scenario

Running an UPDATE or DELETE statement without a proper WHERE clause can affect every row in the table. 

DELETE FROM Customers
-- WHERE CustomerID = 12
Common mistake: Highlighting only part of the query or forgetting to double-check the filter before running it.
