Why “10” is not equal to 10

user_input = input("Enter a number: ")

if user_input == 10:
    print("Correct")
else:
    print("Wrong")

What happened? Input returns a string, not an integer. That means "10" and 10 are different types.

1. Why input() breaks your comparison

Python Beginner bug

The input() function always returns text. So when you compare it directly to a number, your condition fails even if the user typed the correct value.

Common mistake: Forgetting to convert the input first with int().

2. Why your list keeps remembering values

Python Real-world confusion

Mutable default arguments can reuse the same list between function calls. It feels like Python is “remembering” old data because it kind of is.

Common mistake: Using a list as a default parameter instead of starting with None.

6. Why 'is' works for 256 but not 257

Python Hidden behavior

This looks inconsistent — but it's actually because Python reuses small integers in memory, so is behaves differently depending on the value.

Common mistake: Using is to compare values instead of ==.

7. Why modifying one list changes the other

Python Memory trap

It looks like b is a separate list — but it's actually pointing to the same object as a, so changes affect both.

Common mistake: Thinking b = a creates a copy instead of a reference.

8. Why removing items in a loop breaks your list

Python Loop trap

Removing items while looping through a list can skip elements because the list shifts during iteration.

Common mistake: Modifying a list while iterating over it.

9. Why length stays 3 even after clearing the list

Python State trap

The length is calculated before the list is cleared — so the variable keeps the old value even after the list changes.

Common mistake: Thinking variables update automatically when the data changes.

10. Why += changes the original list

Python Mutation trap

It looks harmless, but += can modify the same list in place — which means other variables pointing to it also see the change.

Common mistake: Thinking += always creates a brand new list.

Python Indexing

Python lets you access elements from the end of a list using negative indexes — which is powerful once you understand it.

Common mistake: Thinking negative indexes are invalid or will cause an error.

1. Why input() breaks your comparison

The input() function always returns text. So if you compare it directly to a number, the condition can fail or even throw an error depending on the operator you use.

example.py
example.py
age = input("Enter age: ")

if age >= 18:
    print("Adult")
Common mistake: Forgetting to convert the input first with int().

Better version

better_version.py
better_version.py
age = int(input("Enter age: "))

if age >= 18:
    print("Adult")
  • input() always returns a string.
  • Comparing string to int causes problems.
  • Converting early keeps the rest of your logic clean.
